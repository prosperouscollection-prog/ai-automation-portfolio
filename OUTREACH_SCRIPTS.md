If it ever becomes a priority to improve response time, reduce missed leads, or automate repetitive admin work, feel free to reach out. Happy to be a resource down the line.

### 2. "We already have someone for that"

That makes sense.

In a lot of cases I am not replacing an existing person or team, I am helping automate the repetitive first layer so your existing people can focus on higher-value work.

If useful, I’m happy to send one [Project Name] example just so you can see what I mean.

### 3. "How much does it cost?"

It depends on which workflow makes the most sense, but most of my projects fall between $500 and $5,000 for setup, plus an optional monthly support retainer.

If you want, I can tell you exactly what I’d recommend for [Business Name] and what it would cost based on your use case.

### 4. "I need to think about it"

Absolutely.

I’ll send a short summary so you have something concrete to review, and then I’ll follow up on [Day / Date] in case you have questions.

### 5. "Can you send me more information?"

Definitely.

I can send over a short overview and a relevant [Project Name] demo so you can see what the system looks like in practice. If it seems useful after that, we can schedule a quick call.

### 6. "We don't have the budget right now"

Understood.

In a lot of cases the best place to start is the smallest system tied to the clearest ROI, something that helps recover missed leads or save staff time quickly. If it helps, I can suggest a lighter version that is more budget-friendly.

### 7. "I'm too busy right now"

I get it.

That is usually exactly when automation becomes useful. I can send a short [Project Name] demo you can review in a couple minutes whenever you have time, and if it is interesting we can talk later.

## Best Practices

- Keep the first message short
- Lead with the business problem, not the technology
- Offer a demo, not a hard pitch
- Personalize the first line whenever possible
- Follow up consistently without sounding desperate
- Stop chasing once the signal is clearly cold
- Focus on one pain point per message
